# Assessment Map

## Assessment philosophy

This program is project-heavy because OpenClaw competence is operational. Learners must show they can reason, configure, validate, and defend real setups.

## Assessment categories

### 1. Concept checks

- short quizzes
- architecture diagrams
- config-reading exercises
- threat-analysis prompts

### 2. Practical labs

- environment setup
- install and onboarding
- provider and channel setup
- sandboxing and approvals
- remote access and troubleshooting

### 3. Design reviews

- secure baseline review
- production architecture review
- channel policy review
- plugin design review

### 4. Capstones

- track-specific, defended verbally and with artifacts

## Semester weighting recommendation

### Semester 1

- 20% concept checks
- 50% labs
- 15% design review
- 15% practical exam or mini-project

### Semester 2

- 15% concept checks
- 35% labs
- 20% design reviews
- 30% capstone

## Required assessment artifacts

- one environment verification submission
- one install/onboard/troubleshooting runbook
- one secure baseline configuration review
- one remote or persistent deployment design
- one detached-work or automation authority review
- one track capstone
- one final defense presentation

## Failure conditions

Learners should not pass advanced stages if they cannot:

- explain the official OpenClaw trust model
- distinguish safe loopback-first deployment from unsafe exposure
- use doctor, status, and security audit effectively
- explain why release notes and update channels matter for provider and security-sensitive labs
- explain why tool-enabled agents require explicit governance
