# Submission Templates

## Purpose

These templates reduce variation in student submissions and make grading more consistent across instructors, sections, and cohorts.

## Included templates

- [Lab Submission Template](lab-submission-template.md)
- [Runbook Template](runbook-template.md)
- [Design Review Template](design-review-template.md)
- [Threat Model Template](threat-model-template.md)
- [Capstone Submission Template](capstone-submission-template.md)
- [Release-Aware Note Template](release-aware-note-template.md)

## Recommended usage

### Use for labs

- [Lab Submission Template](lab-submission-template.md)
- [Release-Aware Note Template](release-aware-note-template.md) when the lab touches providers, security, plugins, automation, or deployment

### Use for design reviews

- [Design Review Template](design-review-template.md)
- [Threat Model Template](threat-model-template.md) when the design widens authority, exposure, or automation

### Use for operations work

- [Runbook Template](runbook-template.md)

### Use for final projects

- [Capstone Submission Template](capstone-submission-template.md)

## Instruction rule

Students should be allowed to adapt structure for clarity, but they should not remove:

- evidence sections
- trust-boundary sections
- failure-mode sections
- release-aware sections where relevant
