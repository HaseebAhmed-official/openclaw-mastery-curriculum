# Model Artifacts

## Purpose

These examples show what "good" looks like in this curriculum without pretending there is only one acceptable wording or one perfect structure.

Use them to:

- calibrate students before submission
- calibrate graders before large marking cycles
- distinguish strong operational thinking from polished vagueness

## Included examples

- [Model Lab Submission](model-lab-submission.md)
- [Model Design Review](model-design-review.md)
- [Model Capstone Summary](model-capstone-summary.md)
- [Instructor Model Feedback](instructor-model-feedback.md)
- [Weak Submission Red Flags](weak-submission-red-flags.md)

## Usage rule

These are exemplars, not answer keys to be copied. Instructors should encourage students to match the quality bar, not the exact phrasing.
