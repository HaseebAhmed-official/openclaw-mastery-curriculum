# Slide Deck Outlines

## Purpose

These outlines turn the semester maps into teachable lecture decks. They are designed to save instructor time while keeping the course aligned with the curriculum's technical and security standards.

## Included assets

- [Semester 1 Weekly Slide Outlines](semester-1-weekly-outlines.md)
- [Semester 2 Weekly Slide Outlines](semester-2-weekly-outlines.md)
- [Track Workshop Slide Outlines](track-workshop-outlines.md)

## Deck rules

- every deck should open with why the topic matters operationally
- every deck should include at least one trust, security, or governance checkpoint
- every deck should bridge directly into the week's lab or assessment
- every deck should end with an exit check that proves understanding, not recall
- production and security decks should name the release baseline and sources checked

## Recommended delivery pattern

1. Opening problem or scenario
2. Concept map
3. Architecture or control surface explanation
4. Demo or worked example
5. Failure modes and anti-patterns
6. Lab handoff
7. Exit ticket
