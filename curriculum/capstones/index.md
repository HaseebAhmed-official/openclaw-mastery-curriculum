# Capstones

## Capstone philosophy

Capstones should prove that the learner can make and defend decisions under realistic constraints.

## Core rule

Capstones are track-specific. One shared project is not enough because OpenClaw mastery differs across operators, platform engineers, security practitioners, and extension developers.

## Capstone guide

- [Capstone Specs](capstone-specs.md)

## Capstone evaluation pillars

- technical correctness
- security and trust-boundary reasoning
- operational maturity
- documentation quality
- ability to defend tradeoffs under questioning
