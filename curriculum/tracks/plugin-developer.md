# Plugin Developer Track

## Goal

Produce developers who can extend OpenClaw with well-structured, schema-backed, operationally sane plugins and skills.

## Best fit for

- application developers
- integration engineers
- ecosystem contributors

## Additional emphasis

- plugin architecture
- plugin bundles and distribution surfaces
- manifest design
- JSON Schema validation
- skills and six-layer precedence
- ClawHub ecosystem realities, install/update behavior, and compatibility gating
- capability ownership and config surfaces

## Extra labs

- manifest inspection lab
- config-schema design drill
- skill and plugin interoperability exercise
- plugin install-risk and compatibility exercise

## Capstone

Build a plugin or extension package with manifest, schema, config documentation, install/update notes, compatibility assumptions, and a validation/test plan.
