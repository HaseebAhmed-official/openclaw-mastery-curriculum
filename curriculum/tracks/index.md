# Tracks

## Purpose

Tracks let institutions preserve one rigorous core while still developing depth in specific roles.

Each track:

- reuses the same foundational semesters
- adds focused labs and readings
- ends with a distinct capstone
- maps to a different mastery ceiling

## Track selection guidance

### Choose Operator if you want to:

- run OpenClaw well
- support non-developer users
- focus on workflows, channels, and operational fluency

### Choose Production / DevOps if you want to:

- deploy OpenClaw in persistent environments
- design remote-access and infrastructure patterns
- own uptime, hardening, and change control

### Choose Security / Hardening if you want to:

- evaluate risks and trust boundaries
- design safer policies and response playbooks
- audit operational setups

### Choose Plugin Developer if you want to:

- extend OpenClaw capabilities
- build manifests, schemas, and plugin-driven integrations

### Choose Contributor / Core if you want to:

- work at repo and architecture level
- understand internal surfaces, testing, and contribution norms

### Choose Local Models if you want to:

- reason deeply about self-hosted or on-prem model strategies
- evaluate cost, quality, latency, and safety tradeoffs
